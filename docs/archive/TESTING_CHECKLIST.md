# اختبار شامل لميزات المرحلة 9 و 10

## المرحلة 9: Breadcrumb و SkeletonCard

### 1. اختبار Breadcrumb Navigation

#### Dashboard Page
- [ ] تظهر Breadcrumb في أعلى الصفحة: "Dashboard"
- [ ] تظهر أيقونة المنزل قبل النص
- [ ] يمكن النقر على Breadcrumb للعودة للصفحة السابقة
- [ ] Breadcrumb يظهر بشكل صحيح على الأجهزة المحمولة

#### Calendar Page
- [ ] تظهر Breadcrumb: "Dashboard → Calendar"
- [ ] يمكن النقر على "Dashboard" للعودة
- [ ] يمكن النقر على "Calendar" للبقاء في نفس الصفحة

#### Telegram Posts Page
- [ ] تظهر Breadcrumb: "Dashboard → Telegram Posts"
- [ ] الروابط تعمل بشكل صحيح
- [ ] النص مرئي بوضوح

### 2. اختبار SkeletonCard Loading States

#### Dashboard Page
- [ ] عند تحميل الإحصائيات، تظهر SkeletonCard بدلاً من spinner
- [ ] تحتوي على animation smooth
- [ ] تختفي عند انتهاء التحميل
- [ ] تظهر البيانات الفعلية بعد التحميل

#### Calendar Page
- [ ] عند تحميل الأحداث، تظهر SkeletonCard
- [ ] عدد الـ skeletons يطابق عدد العناصر المتوقعة
- [ ] الـ animation سلس وليست مزعجة

#### Telegram Posts Page
- [ ] عند تحميل المنشورات، تظهر SkeletonCard
- [ ] تظهر بتخطيط مشابه للمنشورات الفعلية
- [ ] الانتقال من skeleton إلى محتوى حقيقي سلس

## المرحلة 10: Undo/Redo و Dark Mode

### 3. اختبار Undo/Redo Functionality

#### في صفحة Generate

**اختبار الكتابة والتراجع:**
- [ ] عند إنشاء محتوى، تظهر أزرار Undo/Redo
- [ ] زر Undo مفعل عند وجود تاريخ للتراجع
- [ ] زر Redo معطل في البداية
- [ ] النقر على Undo يعود للنص السابق
- [ ] النقر على Redo يعود للنص التالي

**اختبار اختصارات لوحة المفاتيح:**
- [ ] Ctrl+Z يعمل للتراجع
- [ ] Ctrl+Shift+Z يعمل للإعادة
- [ ] Ctrl+Y يعمل للإعادة (بديل)
- [ ] الاختصارات تعمل على Mac (Cmd بدلاً من Ctrl)

**اختبار السلوك المتقدم:**
- [ ] عند التراجع عدة خطوات ثم كتابة نص جديد، يتم حذف التاريخ المستقبلي
- [ ] لا يمكن التراجع أكثر من اللازم
- [ ] لا يمكن الإعادة أكثر من اللازم
- [ ] الأزرار تظهر/تختفي بناءً على الحالة

**اختبار التحسينات:**
- [ ] عند تحسين المحتوى (Grammar, Engagement, Clarity)، يتم حفظ النسخة الجديدة في التاريخ
- [ ] يمكن التراجع عن التحسينات
- [ ] يمكن الإعادة بعد التراجع

### 4. اختبار Dark Mode Toggle

#### في Sidebar

**اختبار التبديل:**
- [ ] يوجد زر Dark Mode Toggle في أسفل الـ sidebar
- [ ] النقر على الزر يبدل بين الوضع الفاتح والداكن
- [ ] الأيقونة تتغير (Moon/Sun)
- [ ] التبديل يحدث بسلاسة بدون تأخير

**اختبار الألوان:**
- [ ] في الوضع الفاتح، الخلفية بيضاء والنصوص سوداء
- [ ] في الوضع الداكن، الخلفية سوداء والنصوص بيضاء
- [ ] جميع الألوان قابلة للقراءة في كلا الوضعين
- [ ] الأزرار والعناصر التفاعلية واضحة

**اختبار الـ localStorage:**
- [ ] عند تبديل الوضع وتحديث الصفحة، يبقى الوضع المختار
- [ ] عند إغلاق المتصفح وفتحه مجدداً، يبقى الوضع المختار
- [ ] عند حذف localStorage، يعود للوضع الافتراضي

**اختبار التطبيق على جميع الصفحات:**
- [ ] Dashboard يتبع الوضع المختار
- [ ] Generate page يتبع الوضع المختار
- [ ] Calendar يتبع الوضع المختار
- [ ] Telegram Posts يتبع الوضع المختار
- [ ] Settings يتبع الوضع المختار
